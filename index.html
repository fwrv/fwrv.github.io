Hello world... this is my first website!

Welcome to the Big Lake Club

Over the course of Winter 2015 / 2016, I'm going to be using this site as a place to put my rudimentary html, css and js to the test. Hopefully, it also proves a worthwhile reason for me to go out and learn more.

It's very easy now to take a course on codeacademy or udacity. While this may be an effective way for others to learn how applications and code work (they are certainly great resources), I have found that these courses, taken alone, do a bit too much "hand-holding" to be useful to me. I certainly need some hand-holding, but I don't think I'll be able to truly understand how programming works until I've made something myself.

What is this site for?

I'll be using this site as a place to share several things:

  • My ongoing adventures in the world of programming / app developing, which fall under the category of "What I'm making"
  • Things I've read that I find interesting, or "What I'm reading"
  • Food I'm eating, or "What I'm eating"

What I'm making

Domain name

First step was to find a domain name. Growing up, my family used to rent a cabin in Chocorua, New Hampshire for a couple of weeks every summer. Mt. Chocorua is an incredibly photogenic peak in the White Mountains of northern New England. At the foot of the mountain is Lake Chocorua. A small wood bridge, wide enough for one car and maybe 15 feet in length, divides the lake in two: a "big lake" and a small one. You can probably see where this is going... our family referred to the two as the "big lake" and "small lake." It may be harder to see why all this made me think biglake was a cool domain name, but I think it'll certainly do.

I haven't quite figured out the ".club" part yet... It was my favorite of the ones available on godaddy.com. If you're reading this, you're in the club.

Hosting

Linking in the CSS

This took a fair amount of tinkering and cursing. I didn't quite understand how to link to other files saved locally but in different folders.

Learning CSS

This is clearly an ongoing process that very clever people have turned into an art. Here I actually did take the first 2/3 of a quick Intro to HTML and CSS course on Udacity. This taught me some fundamentals around box / grid structure of websites and some cool tricks around optimising across platforms, etc. CSS is clearly a bit of a rabbit hole though and I want to start playing around in javascript.

AngularJS

Following the hearty recommendations of several friends, I'm learning Angular JS as a way to incorporate some javescript into this app (aka this page - reader's note: I have learned to call this website an "app" or a "web app". Apparently all websites are indeeds applications.) Angular JS is a much-heralded project from google that allows developers to incorporate javescript seemlessly into html applications. As of me adding this text, biglake.club now incoporates Angular... it just isn't using it much yet.

Linking within the biglake.club domain

Ultimately I want this site (/"app") to be able to show more than one screen (i.e. I want people to be able to click around in it and be taken to different content). After consulting my lead consultant Lucas, I learned this is actually pretty straightforward. By adding a new .html file to the github repositiory and putting a simple "a href" link in the text, I can add essentially different versions of the same page that the visitor can visit / "surf to". You'll see the link on "What I'm making" header above does just that, as does THIS text. (Right now the link doesn't take you anywhere very impressive, but I want to build out the core index.html as well as possible before copying so that future changes won't require me duplicating too much work.)

I'd never heard of REDEF, the traditional "distributor" before, nor will I necessarily look directly to that distributor again. Is the real distributor here Azeem, the reader that recommended the piece to him, or me, the person who has shared the content now with you, the audience (i.e. the friend or family member humoring me by checking out this site)? No one involved has had any relationship with the "creatives" of the content (which, at least in my case, is a shame, since they seem like clever guys). While this phenomenon is potentially exciting for them, it also highlights their lack of control. In any case, plenty more to dig into here.
